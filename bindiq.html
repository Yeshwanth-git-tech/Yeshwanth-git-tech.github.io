Architecture

How it works

Six specialized agents operate in a directed graph, each with a discrete responsibility.

📥
Intake Parser
Extracts structured entities from unstructured broker submissions using Claude's extraction pipeline. Identifies carrier, policy type, coverage requirements, and applicant metadata.
Agent 01 · LLM extraction
🔬
Risk Classifier
Traverses the Neo4j knowledge graph to score applicant risk. Correlates business type, claims history, and carrier appetite across 380+ relationship edges.
Agent 02 · Neo4j traversal
🌐
Portal Navigator
Playwright-based browser agent authenticates into live carrier portals — NEXT, Chubb, Simply Business — and retrieves real-time policy and eligibility data.
Agent 03 · Playwright automation
🚨
Denial Detector
Scans retrieved carrier data against 28 payer rules to flag denial codes, missing documents, and billing order violations before submission.
Agent 04 · Rule engine
📋
Quote Generator
Assembles a payer-ready ACORD 125/126 quote package with premium breakdown, coverage terms, and validity window. Formatted for direct carrier submission.
Agent 05 · ACORD formatting
✅
Bind Orchestrator
Submits the final bind request, captures reference numbers, updates the broker workflow state, and triggers notifications. The human reviews; the agent handles the rest.
Agent 06 · LangGraph orchestration
